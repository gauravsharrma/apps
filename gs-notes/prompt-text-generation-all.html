13. Summarization (condensing documents or text)

Summarization is a key task for prompt engineering, with specific strategies outlined. For long documents, techniques like Recursive Summarization and Map-Reduce style summarization are mentioned.

  • Objective: To condense lengthy text into a shorter version while preserving key information and meaning.
  • Relevant Prompt Elements: Instructions, Input Data (the text to summarize), Output Indicator/Format, Tone, Style, Audience, Constraints (length, focus). Context can include the purpose of the summary.
  • Prompting Strategies:
    • Clearly state the instruction: "Summarize the following text...".
    • Provide the text to be summarized as Input Data. Delimiters can be used to separate the instruction from the text.
    • Specify the desired length and format (e.g., "in three bullet points," "into exactly three sentences," "one-paragraph summary"). Length parameters are critical constraints.
    • Define the focus or aspects to include (e.g., "focusing on the main challenges discussed," "focusing on the main event and its immediate impact," "summarize... methodology section").
    • Specify the target audience, as this influences the level of detail and terminology (e.g., "for a general audience," "understandable by a non-technical manager," "suitable for a technical audience").
    • Implicitly or explicitly guide whether the summary should be extractive (using phrases directly from the source) or abstractive (rephrasing). Prompts like "Rewrite and summarize..." encourage abstraction.
    • For very long documents, use techniques like breaking text into chunks, summarizing each chunk, and then synthesizing the chunk summaries into a final summary. Provide context about how the summary will be used (e.g., "The summary will be combined with other summaries...").
  • Example Prompt Framework (Combining strategies from): "Instruction: Summarize the following article. Audience: Technical audience familiar with the subject matter. Format & Length: Provide the summary as a single paragraph, no more than 500 words, suitable for an abstract. 16. Generating Reasoning Steps/Answers (Complex Reasoning Tasks)

This encompasses techniques like Chain-of-Thought (CoT), Self-Consistency, Tree-of-Thoughts (ToT), and Self-Ask, which are designed to improve the AI's ability to handle complex problems and logic.

  • Objective: To guide the AI through explicit reasoning steps to solve complex problems, improve accuracy, and make the process transparent.
  • Relevant Prompt Elements: Instructions, Input Data (the problem), Context, Output Indicator/Format (showing steps).
  • Prompting Strategies:
    • Chain-of-Thought (CoT): Instruct the AI to generate intermediate reasoning steps before providing the final answer. Use phrases like "think step by step" or "let's work through this". Break complex problems into smaller parts and ask for intermediate conclusions. CoT elicits reasoning by decomposing the problem.
    • Few-Shot CoT: Provide examples of the problem-solving process, including the step-by-step reasoning leading to the correct answer.
    • Zero-Shot CoT: Simply add "Let's think step by step" or a similar phrase to the prompt, relying on the model's inherent ability to generate reasoning steps.
    • Self-Consistency: This is a decoding strategy that works with CoT. It involves generating multiple diverse reasoning paths for the same problem and selecting the most consistent answer (often via majority vote). To implement, you'd typically use a CoT prompt and configure the model to sample multiple outputs. The prompt itself doesn't necessarily change, but the model configuration and subsequent processing of outputs do.
    • Tree-of-Thoughts (ToT): For very complex problems, guide the AI to explore multiple different possibilities or "thoughts" at each step, evaluate their promise, and select the best path. This can involve prompting for thought generation ("generate 3 distinct possible next steps") and state evaluation ("Evaluate the following thought... Rate its potential...").
    • Self-Ask: Prompt the model to first rephrase and expand upon the question, then break it down into simpler, sequential sub-questions, answer them, and finally synthesize the overall answer. This helps clarify ambiguity and structure reasoning. An example structure is "Question: [Your complex question] Are follow up questions needed here: Yes. Follow up: [Sub-question 1] Intermediate answer: [...] Follow up: [Sub-question 2] Intermediate answer: [...] So the final answer is: [...]".
  • Example Prompt Framework (Zero-Shot CoT Example from sources and Self-Ask example from):
    • Zero-Shot CoT: "Question: Natalia sold clips to 48 of her friends and then found 7 more. If she had 24 clips at first, how many did she have at the end? Instruction: Let's think step by step."
    • Self-Ask: "Instruction: Answer the following question by first asking and answering necessary follow-up questions step-by-step. Question: Who lived longer, Theodor Haecker or Harry Vaughan Watkins?"
  • Mechanism: These techniques guide the LLM's internal process, encouraging it to activate step-by-step reasoning and explore possibilities rather than just generating a direct, potentially incorrect, answer. CoT generates intermediate steps, decomposing the problem. Self-Consistency uses multiple reasoning paths for reliability. ToT explores and evaluates multiple potential thoughts. Self-Ask breaks down the query and answers sub-questions sequentially.
